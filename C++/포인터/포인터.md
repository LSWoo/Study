```Text
포인터 : 주소를 저장하는 메모리이다. 

포인터를 사용하는 이유 : 전역을 사용하지않고 스택메모리에 있는 변수(지역변수)를 다른 함수에서 수정하기위해서 
```
### 포인터 사용 예제
- 포인터의 선언 방법 : TYPE* 변수이름;

```C++
int value = 10;
int* num; 

num = &value;
// value 의 주소값(&value) 를 num 에 저장한다. 

int result = *num; 
*num = 2;
```


## 값 복사 방식
```Text
C++ 에서 구조체를 복사하는 과정을 어셈블리로 보게 되면 변수 한개마다 복사가 일어나는걸 확인할 수 있다. 
```
```C++
StatInfo CretePlayer();

struct StatInfo
{
  int hp;
  int attack;
  int defence;
}

int main()
{
  Statinfo PlayerInfo;
  
  PlayerInfo = CreatePlayer();
  
  return 0;
}

StatInfo CretePlayer()
{
  StatInfo resultValue;
  
  resultValue.hp = 100;
  resultValue.attack = 10;
  resultValue.defence = 5;
  
  return resultValue;
}
```

## 주소값 전달 방식
```Text
값 전달 방식보다 주소값 전달방식의 장점은 값을 복사 하지않아 불필요한 메모리 사용을 줄일수있다.
```

```C++
void CretePlayer();

struct StatInfo
{
  int hp;
  int attack;
  int defence;
}

int main()
{
  Statinfo PlayerInfo;

  CreatePlayer(&PlayerInfo);
  
  return 0;
}

void CretePlayer(StatInfo* info)
{
  info->hp = 100;
  info->attack = 10;
  info->defence = 5;
}
```

## 포인터 참조 방식
```Text
주소값 전달 방식가 성능면에서는 완전히 동일하고 사용하기에 더 편리하지만 값 복사 방식과 헷갈릴수 있다는 단점이 있다.
```

```C++
void CretePlayer();

struct StatInfo
{
  int hp;
  int attack;
  int defence;
}

int main()
{
  Statinfo PlayerInfo;

  CreatePlayer(PlayerInfo);
  
  return 0;
}

void CretePlayer(StatInfo& info)
{
  info.hp = 100;
  info.attack = 10;
  info.defence = 5;
}
```
